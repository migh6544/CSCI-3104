\section{Standard 20- Quicksort: Analysis}
\begin{required}
Assume that at each partitioning of a subarray of size $n$, you always get two subarrays of size $10$ and size $n-11$, respectively. [Note that the pivot is not in either sub-array, so $10$ and $n-11$ are indeed correct.] What would be running time of the resulting QuickSort algorithm? Use a suitable asymptotic notation to represent the running time, and {\bf explain/justify} your answer.
\end{required}

\begin{proof}[Answer:] \
\begin{align*}
T(n) &\leq T(10) + T(n - 11) + cn \\
&\leq T(10) + [T(10) + T(n - 22) + c(n - 11)] + cn \\
&= 2T(10) + c[n + (n - 11)] + T(n - 22) \\
&\leq 2T(10) + c[n + (n - 11)] + T(10) + T(n - 33) + c(n - 22) \\
&= 3T(10) + c[n + (n - 11) + (n - 22)] + T(n - 33) \\
&\leq kT(10) + c[\sum_{i = 0}^{k} (n - 11i)] + T(n - 11k) \\
\end{align*}
\item \textbf{Base case reached when $n - 11k \leq 1$}
\item i.e $k \approx \frac{n}{11}$ \\
\item Thus:
\item $\frac{n}{11} \cdot T(10) + c11\sum_{i = 0}^{\frac{n}{11}} i$
\item = $c^{\prime}n + c11\Theta(\frac{n}{11})^{2}$
\item = $\Theta(n^{2})$ \\
\item \textbf{So \color{red}$T(n) \in \Theta(n^{2})$}

%Your answer here
\end{proof}
